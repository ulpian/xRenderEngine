# Glyph calibration (glyphgen)

`glyphgen` is the offline dev tool that turns a font into rendering data: a
**luminance ramp** and, optionally, a table of **6-region shape vectors**. It is
the `glyphgen` crate under `tools/`, exposed both as a standalone `glyphgen` binary
and as the [`xre glyphgen`](cli.md) subcommand.

## Characters are not pixels

A pixel has one property: brightness. A terminal cell has a *glyph*, and glyphs
differ wildly in how much ink they put on screen — and *where*. A `.` is mostly
empty and bottom-weighted; a `@` is nearly solid; a `'` lives at the top. To pick
the right glyph for a given coverage, the renderer needs to know each glyph's actual
ink, and that number is **font-dependent**: Menlo's `#` is not JetBrains Mono's `#`.
Calibration measures it once, offline, so the runtime never touches a font file.

## What it computes

Two measurements, both by rasterizing each glyph with `ab_glyph` and summing
anti-aliased coverage:

- **Luminance ramp.** Each glyph's summed ink is divided by its cell area and
  normalized so the densest glyph is `1.0`. Entries are sorted ascending and
  de-duplicated by a visual-distance threshold (`min_gap`) so the ramp stays crisp.
  Output: `[(coverage, glyph)]`.
- **Shape vectors.** Each glyph's ink is binned into a 2×3 region grid —
  `[top-left, top-right, mid-left, mid-right, bottom-left, bottom-right]` — and
  normalized **per component** across the set. Output: `[(glyph, [f32; 6])]`. This
  is what lets the [shape shader](cell-shaders.md) tell a top-heavy cell from a
  bottom-heavy one.

When no font is available, the tool emits a font-free, density-ordered ASCII ramp
(the `DENSITY_ORDER` string) with uniform coverage spacing — useful for
bootstrapping.

## Running it

The CLI emits a **luminance ramp** as Rust source. With a font:

```sh
xre glyphgen --font Menlo.ttf --out assets/atlas_menlo.rs
# or the standalone binary:
glyphgen --font Menlo.ttf --out assets/atlas_menlo.rs
```

Without a font, you get the built-in density ramp:

```sh
xre glyphgen --builtin --out assets/atlas_generic.rs
```

Flags (all optional):

```text
--font <ttf>      measure this TTF/OTF; omit for the built-in ramp
--builtin         force the font-free density ramp
--out <atlas.rs>  write here; without it, source goes to stdout
--name <IDENT>    static name (default: RAMP_<FONTSTEM> or RAMP_GENERIC)
--px <f32>        rasterization size (default 64.0)
--charset <str>   glyphs to measure (default: the 95 printable ASCII)
--min-gap <f32>   coverage-distance dedupe threshold (default 0.004)
-h, --help        usage
```

The emitted file is a single static, meant to be checked in and `include!`d —
calibration is data, never logic:

```rust,ignore
//! Auto-generated by glyphgen. Do not edit by hand.
pub static RAMP_MENLO: &[(f32, char)] = &[
    (0.0000, ' '),
    (0.0123, '.'),
    // ...
    (1.0000, '$'),
];
```

## Feeding the output into the renderer

Pass the generated ramp straight to the luminance shader:

```rust,ignore
use xre::render::LuminanceRamp;

let ramp = LuminanceRamp::from_ramp(RAMP_MENLO.to_vec());
```

`from_ramp` re-sorts entries by coverage ascending, then does an `O(log n)`
nearest-neighbour select per cell. (The default `LuminanceRamp` uses
`from_density(DENSITY_ORDER)`, so the engine renders with no atlas at all.)

For directional shading, build a `ShapeTable` from a shape table and wrap it in a
`ShapeVector`:

```rust,ignore
use xre::render::{ShapeGlyph, ShapeTable, ShapeVector};

let glyphs = SHAPES_MENLO
    .iter()
    .map(|&(glyph, vector)| ShapeGlyph { glyph, vector })
    .collect();
let table = ShapeTable::new(glyphs); // re-normalizes per component
let shader = ShapeVector::new(table);
```

`ShapeTable::new` normalizes each of the six components across the set — the
critical fix without which lookups collapse onto border glyphs. The built-in
`ShapeTable::builtin_ascii()` is hand-authored and ships by default; a font-measured
table is sharper.

> The CLI currently wires the **ramp** path. Shape-vector generation is available
> as a library function for build scripts and tests that want to bake a
> font-measured `ShapeTable`.

## Licensing

`glyphgen` depends on `ab_glyph` to read font outlines, and that dependency **never
ships in the engine** — it lives only in this dev tool. The artifacts you commit are
*measurements* (coverage floats and char literals), not font outlines, so generated
ramps and shape tables carry no font data. Always confirm a font's license permits
the measurements you redistribute; when in doubt, ship the `--builtin` ramp, which
is font-free.
